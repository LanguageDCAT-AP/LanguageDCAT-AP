Abstract

LanguageDCAT-AP is conceived and implemented as an extension of (a) the DCAT-AP application profile for Catalogues containing Datasets and Data Services in Europe [[DCAT-AP]] and (b) the MLDCAT-AP profile for machine learning models [MLDCAT-AP], catering specifically for the language data, language models and language technolgy services domain. To this end, it adopts classes and properties from the META-SHARE ontology, which is already used in the Language Resources and Technologies community [MS-OWL], introduces new classes and properties and proposes domain-specific controlled vocabularies.

Work on the LanguageDCAT-AP started in the context of the Common European Language Data Space but it is designed with a wider scope covering broader description needs in the domain.

Terminology

An Application Profile is a data specification aimed to facilitate the data exchange in a well-defined application context. It re-uses concepts from one or more semantic data specifications, while adding more specificity, by identifying mandatory, recommended, and optional elements, addressing particular application needs, and providing recommendations for controlled vocabularies to be used. More information can be found on the SEMIC Style Guide. TO ADD language resoruce, language data, language models, language technology tool/service
